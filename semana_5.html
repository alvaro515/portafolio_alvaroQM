Teoría y Conceptos de la Semana

1. Estructura y Operación de un Framework Front-End

Un **framework front-end** es un conjunto de herramientas y bibliotecas diseñadas para facilitar la creación de interfaces de usuario en aplicaciones web. Proporcionan una estructura y un flujo de trabajo definidos para desarrollar aplicaciones web eficientes, escalables y fáciles de mantener.

El objetivo principal de un framework front-end es facilitar el desarrollo de interfaces de usuario interactivas, de manera que se pueda concentrar más en la lógica de la aplicación y menos en los detalles de la implementación del UI.

¿Cómo funcionan?

El **Client Side Rendering (CSR)** es una técnica en la que la mayor parte del renderizado de la página se realiza en el navegador, lo que significa que el servidor solo entrega los archivos iniciales y la lógica de la aplicación, mientras que el navegador se encarga de cargar y renderizar el contenido dinámico.

Esto permite una experiencia de usuario más fluida, ya que solo se cargan los datos necesarios y las interacciones no requieren recargar toda la página.

Uno de los frameworks más populares es **React**. En React, la interfaz se divide en componentes, que son piezas reutilizables de la interfaz de usuario.

2. Gestión de Dependencias

En el desarrollo moderno de aplicaciones front-end, utilizamos **gestores de dependencias** como **npm** o **yarn** para instalar y gestionar las bibliotecas y herramientas que necesitamos. Estas herramientas permiten instalar paquetes, actualizar versiones y manejar las dependencias de manera eficiente.

¿Cómo instalar dependencias? Usamos los siguientes comandos:

        
          npm install    // Instalar una dependencia
          npm update     // Actualizar una dependencia

Además, los **paquetes** de npm o yarn se guardan en un archivo llamado **package.json**, que mantiene un registro de todas las dependencias de tu proyecto.

3. Desarrollo Basado en Componentes

En los frameworks modernos como **React**, las interfaces de usuario se construyen utilizando componentes. Un **componente** es una unidad de código autónoma que maneja una parte de la interfaz de usuario. Los componentes pueden ser reutilizados en diferentes partes de la aplicación.

Los **componentes** permiten un desarrollo más organizado y modular. Cada componente tiene su propio estado, lógica y vista.

JSX: JavaScript XML

**JSX** es una extensión de JavaScript utilizada en React para escribir componentes. Aunque parece una mezcla de HTML y JavaScript, JSX es en realidad un azúcar sintáctico que se compila en JavaScript.

Ejemplo de JSX:

        
          const MiComponente = () => {
            return (
              <div>
                <h1>¡Hola Mundo!</h1>
                <p>Este es un ejemplo de JSX.</p>
              </div>
            );
          };

Props y Children

En React, los componentes pueden recibir **props** (abreviatura de "properties") que permiten pasar datos desde el componente padre hacia el componente hijo. Además, los componentes pueden tener **children**, que son elementos anidados dentro de un componente.

Ejemplo de pasar datos con props:

        
          const Saludo = (props) => {
            return <h1>Hola, {props.nombre}!</h1>
          };

          // Usando el componente Saludo
          <Saludo nombre="Juan" />

Ejemplo de children: Los children son lo que se coloca dentro de las etiquetas de un componente, como los elementos hijos de un div:

        
          const Contenedor = (props) => {
            return <div>{props.children}</div>
          };

          // Usando el componente Contenedor
          <Contenedor>Este es el contenido dentro del contenedor.</Contenedor>

4. Estilos en Framework JS

En el desarrollo de aplicaciones web modernas, los estilos pueden ser manejados de diferentes maneras. A continuación, veremos algunas de las formas más comunes de aplicar estilos a los componentes:

Estilos Inline

Los **estilos inline** se aplican directamente dentro de los componentes utilizando la propiedad style en JSX.

        
          const MiComponente = () => {
            return <div style={{color: 'blue', fontSize: '20px'}}>Texto azul y grande</div>
          };

Style Sheets (Hojas de Estilo)

Una forma clásica de agregar estilos es utilizando **hojas de estilo externas** (CSS). Este método es común en proyectos más grandes y se usa cuando los estilos son comunes a varios componentes.

Ejemplo: En un archivo CSS:

        
          .mi-clase {
            color: red;
            font-size: 18px;
          }

Y luego lo importamos en el archivo JavaScript:

        
          import './styles.css';

Style Modules

Los **módulos de estilo** son una técnica para crear estilos específicos para un componente y asegurarse de que no se filtren en otros componentes.

Ejemplo: Definir un archivo de módulo CSS, como MiComponente.module.css:

        
          .miClase {
            color: green;
            font-size: 16px;
          }

Y luego, importamos y aplicamos el estilo en el componente:

        
          import styles from './MiComponente.module.css';

          const MiComponente = () => {
            return <div className={styles.miClase}>Texto verde</div>
          };

Frameworks CSS

Los **frameworks CSS** como **Bootstrap** y **Tailwind CSS** también se pueden usar en aplicaciones React. Estos proporcionan clases predefinidas para facilitar el diseño y la estructura de la interfaz de usuario.
